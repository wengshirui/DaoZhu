# 🎭 话剧导演 · 智能实体剧场

## 项目简介

**话剧导演** 是一个轻量级的 Web 演绎系统，它借鉴话剧的核心理念：**场景 + 实体 + 剧情**。用户只需输入一段自然语言描述（例如：“小明早晨起床了，打开了门”），系统就会自动生成对应的 HTML 页面，并在 Canvas 上以“话剧风格”完成演出。

系统将 AI 作为“导演”，负责解析文本、编排剧本；将画布中的每个物件（人物、门、床等）作为“实体演员”，按照导演的指令完成状态变化、简单移动和文字对白。所有表演均基于预定义的实体能力和场景布置，无需复杂的精灵动画或物理引擎。

## 目标用户

- **独立开发者 / 游戏爱好者**：快速验证“文本 → 动画”想法，搭建原型。
- **教育者 / 故事创作者**：用简单的描述生成互动小剧场，用于教学或创意表达。
- **AI 研究者**：作为测试多智能体编排、LLM 生成结构化数据的实验环境。

## 核心特性

- **自然语言驱动**：用户输入任意短文本，系统自动理解并编排表演。
- **实体演员库**：内置常见角色与道具（人、门、床等），支持扩展。
- **话剧式演绎**：以状态变化 + 文字气泡 + 简易位移为主，不追求复杂动画，专注逻辑清晰。
- **纯前端实现**：最终输出一个独立的 HTML 文件，双击即可运行，无需任何后端或依赖。
- **可重放 / 调试**：提供“重新演绎”按钮，方便反复观察剧本执行。

## 约束与限制

1. **输入文本范围**：当前仅支持非常基础的日常动作描述（起床、走动、开门等）。复杂情节（多角色对话、并行动作、条件分支）暂不支持。
2. **实体能力有限**：每个实体的行为集合需预先定义（如“小明”只能躺、坐、站、走、开门）。无法自动生成全新动作。
3. **视觉风格**：采用极简的“话剧舞台”风格（色块 + 标签 + 文字），不提供像素艺术或高质量精灵图。
4. **并发处理**：剧本按顺序串行执行，不支持同时执行两个动作（如边走边说话）。
5. **无永久存储**：实体库和场景配置均写死在代码中，不支持运行时动态添加新实体。
