# 🎭 AutoMovie · 智能实体剧场

## 项目简介

AutoMovie 是一个轻量级的 Web 动画演绎系统。用户输入一段文本（小说、散文、剧本），AI 自动解析并生成可播放的动画。

**核心理念：Less is More — 用最简单的视觉表达最丰富的故事。**

---

## 演进历程（demo1 → demo6）

| 版本 | 方案 | 结论 |
|------|------|------|
| demo1 | 单文件 Canvas 手绘 | ✅ MVP 验证可行 |
| demo2 | Canvas 精细手绘（表情/木纹/光影） | ✅ 视觉好，但不可扩展 |
| demo3 | 纯 SVG 图标（Lucide/Tabler） | ❌ 图标太简陋，像线框图 |
| demo4 | Canvas 手绘 + SVG 混合 | ❌ 越细节越失真（恐怖谷） |
| demo5 | 极简火柴人 + emoji | ✅ 方向对了！简单但有表达力 |
| **demo6** | **火柴人 + emoji + SVG装饰 + 时间轴** | ✅ **最佳方案** |

## 最终方案（demo6）

### 架构

```
火柴人角色（Canvas）
  + emoji 表情（DOM 弹出动画）
  + SVG 素材装饰（场景氛围）
  + 时间轴驱动（连续动画，无幕切换）
  + 底部字幕条（旁白 + 对话统一位置）
```

### 设计原则

1. **角色 = 颜色 + 火柴人** — 圆头 + 线条身体，颜色区分身份
2. **情感 = emoji** — 头顶弹出 emoji，比画表情更直接
3. **动作 = 姿态 pose** — 6种手臂姿态（normal/up/hip/point/hug/wave）
4. **运动 = 位移 + 物理** — lerp 插值平滑移动，走路时腿部 sin() 摆动
5. **场景 = 渐变背景 + SVG 装饰** — 极简背景 + assets/ 中的 SVG 点缀氛围
6. **叙事 = 底部字幕** — 旁白（深色底）+ 对话（白色底）统一在底部，视线不跳跃
7. **节奏 = 时间轴** — 连续播放，角色自然进出场，不硬切幕

### 文件结构

```
demo6/
├── index.html          # 结构
├── css/style.css       # 样式 + emoji 动画
└── js/play.js          # 引擎（角色 + 背景 + 时间轴 + SVG加载）
```

## 踩坑记录

### ❌ 不要做的

| 坑 | 原因 |
|----|------|
| 追求写实细节 | 越细节越暴露不真实（火车悬空、攀爬不自然） |
| 用 UI 图标当场景实体 | Lucide/Tabler 是 24px 图标，放大后太简陋 |
| 一幕一幕硬切 | 割裂感强，不如连续时间轴流畅 |
| 旁白和对话分散放置 | 观众视线跳跃（左下看旁白、右上看对话） |
| 给每个物体画复杂细节 | 投入产出比极低，不如用符号化表达 |
| 用 CSS transition 做角色移动 | 不够精确，用 Canvas + lerp 更可控 |

### ✅ 应该做的

| 原则 | 做法 |
|------|------|
| 角色辨识 | 颜色 + 名牌，一眼认出 |
| 情感表达 | emoji 弹出 > 画面部表情 |
| 动作表达 | 手臂姿态变化 + 身体抖动 > 复杂骨骼动画 |
| 场景氛围 | 渐变背景 + 少量 SVG 装饰 > 画满细节 |
| 叙事节奏 | 时间轴连续播放 > 分幕切换 |
| 文字信息 | 统一底部字幕条 > 分散气泡 |
| 远近关系 | opacity + scale 变化 > 画透视 |

## 适合的内容类型

| 类型 | 适合度 | 原因 |
|------|--------|------|
| 小说/散文 | ⭐⭐⭐⭐⭐ | 对话多、情感丰富、场景切换 |
| 课文/故事 | ⭐⭐⭐⭐⭐ | 教育场景，简单直观 |
| 剧本 | ⭐⭐⭐⭐ | 天然适合多角色对话 |
| 诗歌 | ⭐⭐⭐ | 意境为主，需要更多视觉氛围 |
| 技术文档 | ⭐⭐ | 不太适合，缺少叙事性 |
