## 原始需求

> 调用工具记录要记录到 db 中，方便以后优化。

## 提取意图

当前工具调用只在内存中追踪（memory_db 的 skill_stats），没有完整的调用日志。需要将每次工具调用的详细信息（工具名、参数、结果、耗时、是否成功）持久化到 SQLite，为后续分析优化提供数据基础。

---

## 开放问题

| # | 模块 | 问题 | AI 猜测 | 答案 |
|---|------|------|---------|------|
| 1 | 数据范围 | 每次调用需要记录哪些字段？工具名、参数、结果全文、耗时、会话ID？ | 全部记录：tool_name, args(JSON), result(JSON), duration_ms, success, conversation_id, timestamp | |
| 2 | 存储策略 | 日志数据量可能很大，是否需要自动清理策略（如保留最近 30 天）？ | 保留 90 天，超期自动清理 | |
| 3 | 使用场景 | 这些数据除了开发者分析，用户是否也能在界面上查看（如"最近操作记录"）？ | 暂时只供开发分析，后续可加 UI | |
