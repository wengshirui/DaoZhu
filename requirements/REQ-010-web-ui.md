## 问题陈述

CLI 适合开发调试，但最终用户需要一个友好的 Web 界面。界面采用左右对称布局：左侧是业务工作区（账套管理、待办提醒、数据展示），右侧是 AI 对话区（新建对话、历史对话、流式交互）。不同于 Hermes 的 PTY 方案，AccoBot 的 Web UI 需要原生支持 Windows，采用标准的 HTTP/WebSocket 通信。

## 用户故事

**As a** 用户  
**I want** 打开浏览器左边看业务数据、右边和 AI 对话  
**So that** 我可以一边查看账务信息一边让 AI 帮我操作，工作流不中断

**三栏布局原则：**
- 左侧（约 20%）：业务工作区 — 账套管理、待办提醒、数据展示
- 中间（约 60%）：当前对话区 — 对话内容（流式输出）+ 底部输入框
- 右侧（约 20%）：对话管理区 — 新建对话按钮 + 历史对话列表（按日期分组）
- 左右两栏可折叠，中间对话区始终可见

## 范围

### In Scope

- **三栏布局** — 左侧业务区（20%）+ 中间对话区（60%）+ 右侧对话管理区（20%），左右可折叠
- **左侧业务区** — 账套管理（切换/新建/删除/打开文件夹）、待办提醒、数据展示（表格/图表）
- **中间对话区** — 当前对话内容（流式输出、Markdown 渲染、工具调用展示）+ 底部输入框
- **右侧对话管理区** — 新建对话按钮 + 历史对话列表（按日期分组，显示标题，点击切换）
- **数据展示** — 表格（账簿/凭证列表）、图表（分析结果）
- **文件上传** — 拖拽上传发票图片、银行流水 Excel
- **响应式布局** — 桌面端三栏，平板端左右栏折叠为抽屉
- **本地启动** — `python -m accobot web` 一键启动，浏览器自动打开

### Out of Scope

- 移动端 App
- 多用户在线协作
- 用户注册/登录系统（单机版不需要）
