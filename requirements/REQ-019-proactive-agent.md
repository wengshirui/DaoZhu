# REQ-019 Agent 主动任务机制

**状态：** 📝 Draft  
**优先级：** M  
**来源：** 2025-05-21 架构 Grill 讨论

---

## 问题陈述

当前 Heartbeat 仅做被动检查并推送文字通知，无法让 Agent 主动执行检测、配置等操作。财务场景天然需要主动提醒（报税截止、月末结转、内控检查），且首次启动时需要 Agent 自主完成环境初始化。需要一个通用的"Agent 主动任务"机制来统一处理这些场景。

## 范围

**In Scope:**
- 通用的主动任务注册与调度框架
- 任务分类：一次性（首次启动）vs 周期性（定时检查）
- 任务结果统一推送到 Web UI 待办面板
- 所有写操作必须经用户确认后才执行（通知级，非自动级）

**Out of Scope:**
- 多用户并发（单用户产品）
- 自动执行写操作（0.1 版本不做）
- 复杂的任务依赖编排

## 用户故事

As a 小微企业老板，I want AccoBot 能主动发现问题并提醒我，so that 我不会错过报税截止日、忘记审核凭证、或遗漏环境配置。

## 验收标准

1. **AC-1 任务注册**：系统支持注册主动任务，每个任务包含：名称、触发条件（定时/一次性/条件触发）、检查逻辑、结果处理方式
2. **AC-2 通知推送**：任务检测到需要用户关注的事项时，通过 WebSocket 推送到 Web UI 的待办面板，包含类型标识和可读消息
3. **AC-3 用户确认流**：待办项支持"确认执行"操作，用户点击确认后 Agent 才执行对应的写操作
4. **AC-4 一次性任务**：支持标记任务为"完成后不再执行"，用持久化标记（如 config 文件）记录完成状态
5. **AC-5 错误隔离**：单个任务执行失败不影响其他任务和主对话流程

## 业务价值

- 减少用户遗忘风险：报税逾期罚款（小规模纳税人每次 200-2000 元）
- 提升产品粘性：用户每天打开就能看到待办，形成使用习惯
- 为后续内控审计功能打基础

## T-Shirt Size

**M** - 需要新增任务调度框架 + 改造现有 Heartbeat + 前端待办面板交互；风险在于任务执行与主对话的并发控制
