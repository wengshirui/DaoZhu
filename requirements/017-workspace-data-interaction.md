# REQ-017: 管家能操作工作区数据

> Intake ID: 2025-05-25-workspace-data-interaction
> 优先级: 🔴 高（核心体验缺失）

---

## 问题陈述

管家现在只能"构建"工作区，不能"使用"工作区。用户说"帮我记一笔账"或"添加一个待办"，管家应该能直接操作对应工作区的数据库，而不是只会说"请打开工作区自己操作"。

## 用户故事

**As a** 岛主用户，
**I want** 通过对话让管家直接操作工作区的数据（如添加待办、记账），
**so that** 我不需要切换到工作区页面，在聊天窗口就能完成日常操作。

## 示例对话

```
用户: 帮我添加一个待办：明天下午开会
管家: ✅ 已添加到「个人待办」：明天下午开会（截止日期：明天）

用户: 记一笔：今天午饭花了 35 元
管家: ✅ 已记录到「财务助手」：管理费用-餐饮 35 元（现金支付）

用户: 我的待办还有哪些没完成？
管家: 你有 3 个未完成的待办：
      1. 明天下午开会
      2. 完成岛主平台开发
      3. 写工作区 README
```

## 验收标准

| # | AC | 可测试性 |
|---|-----|----------|
| 1 | 用户说"添加待办 XXX"，管家调用 todo 工作区 API 创建任务 | 检查 todo/data.db |
| 2 | 用户说"记一笔 XXX"，管家调用 accobot 工作区 API 创建凭证 | 检查 accobot/data.db |
| 3 | 用户说"我的待办有哪些"，管家查询 todo 数据并返回列表 | 对话中显示列表 |
| 4 | 工作区未启动时，管家自动启动后再操作 | 停止 todo 后测试 |

## T-Shirt Size

**M (5 pts)** — 需要给 Agent 注册工作区 API 调用工具，动态发现各工作区的接口
