# REQ-020 首次启动环境初始化

**状态：** 📝 Draft  
**优先级：** S  
**来源：** 2025-05-21 架构 Grill 讨论

---

## 问题陈述

AccoBot 打包为 exe 分发给不懂技术的小微企业老板。运行依赖（Node.js/npx 用于 Playwright MCP）可能未安装。需要 Agent 在首次启动后自主检测环境、安装缺失依赖、完成基础配置，完成后标记为已完成，后续启动跳过。

## 范围

**In Scope:**
- Agent 启动后执行环境预检（Node.js、npx、浏览器可用性）
- 缺失依赖时自动下载安装（静默或最小交互）
- 基础配置引导（API Key 是否已设置、账套是否已创建）
- 完成标记持久化，后续启动跳过

**Out of Scope:**
- 操作系统级别的权限提升（UAC）
- Linux/macOS 支持（0.1 仅 Windows exe）
- 复杂的安装失败恢复

## 用户故事

As a 不懂技术的小微企业老板，I want 双击 exe 后一切自动就绪，so that 我不需要手动安装任何依赖或配置环境。

## 验收标准

1. **AC-1 环境检测**：Agent 启动后检测 Node.js/npx 是否可用，浏览器（Chrome/Edge）是否已安装，结果记录到日志
2. **AC-2 自动安装**：如果 Node.js 未安装，Agent 自动下载并安装 Node.js LTS（Windows msi 静默安装），安装后验证 npx 可用
3. **AC-3 配置引导**：如果 API Key 未设置或账套未创建，在 Web UI 待办面板推送引导提示（非阻塞）
4. **AC-4 完成标记**：环境初始化完成后在 `~/.accobot/` 下写入标记文件（如 `.setup_complete`），后续启动检测到标记则跳过
5. **AC-5 失败降级**：如果自动安装失败（网络问题等），推送通知告知用户手动安装步骤，不阻塞主功能使用（Playwright MCP 标记为不可用即可）

## 业务价值

- 消除技术门槛：目标用户（小微老板）无法自行安装 Node.js
- 减少首次使用流失率：预计可将"安装后放弃"率从 ~40% 降至 <10%
- 浏览器自动化（报税、开票）是核心差异化功能，依赖环境就绪

## T-Shirt Size

**S** - 逻辑简单（检测+下载+静默安装+标记），主要工作量在 Windows 静默安装的调试；风险是不同 Windows 版本的兼容性

## 依赖与约束

- 依赖 REQ-019（作为一次性主动任务注册）
- 依赖 REQ-011（Playwright MCP 是被检测的目标）
- 约束：仅 Windows 平台
- 约束：需要网络连接下载 Node.js（离线场景降级处理）
