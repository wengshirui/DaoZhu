# REQ-014 学习辅导功能

**状态：** 🔀 Merged into REQ-012  
**优先级：** P1（提升用户粘性和成长体验）  
**T-Shirt Size：** M - 知识库结构明确，掌握度追踪有一定设计复杂度

---

## 问题陈述

AccoBot 的目标用户包括不懂会计的老板。他们在使用过程中会遇到大量不理解的概念和规则（什么是借贷？为什么要计提折旧？进项税是什么？）。Bot 不仅要帮他们完成工作，还要在过程中教会他们，让用户越用越懂。

## 用户故事

**As a** 不懂会计的用户  
**I want** 遇到不明白的地方时，AI 记录下来并教我相关知识  
**So that** 我在使用过程中逐渐掌握财务知识，不再是"两眼一抹黑"

## 范围

### In Scope

- **问题自动捕获** — 用户提问或表达困惑时，自动记录到学习区域
- **知识点关联** — 每个问题关联对应的知识点（会计准则条文、税法规定、操作规范）
- **掌握度追踪** — 每个知识点标记用户掌握状态：未学习 / 已了解 / 已掌握
- **智能讲解** — 根据用户掌握程度调整讲解深度（新手用大白话，进阶用专业术语）
- **学习记录查看** — 用户可查看自己的学习进度、薄弱知识点
- **复习提醒** — 长时间未复习的知识点主动提醒
- **上下文教学** — 在实际操作中穿插讲解（"顺便说一下，这里用的是权责发生制，意思是……"）

### Out of Scope

- 系统化课程体系（不是在线教育平台）
- 考试/测验功能
- 证书/学分

## 验收标准

1. **AC-1：问题捕获** — 用户问"什么是进项税"，Agent 回答后自动将"进项税"记录到学习区域，关联知识点"增值税-进项税额"
2. **AC-2：掌握度更新** — 用户第一次问标记为"已了解"，后续操作中正确使用该概念后自动升级为"已掌握"
3. **AC-3：自适应讲解** — 对"未学习"的知识点用通俗比喻解释；对"已了解"的知识点直接用术语，不再啰嗦
4. **AC-4：学习报告** — 用户说"我学了哪些东西"，Agent 返回学习统计（已掌握 X 个、学习中 X 个、待学习 X 个）
5. **AC-5：薄弱提示** — 用户操作涉及未掌握的知识点时，Agent 主动简短提示（不打断工作流）
6. **AC-6：知识点浏览** — 用户说"给我看看关于增值税的知识点"，Agent 返回相关知识点列表及掌握状态

## 数据模型草案

```
learning_records（学习记录）
├── id TEXT PK
├── question TEXT          -- 用户的原始问题
├── knowledge_point_id TEXT -- 关联知识点
├── asked_at REAL          -- 提问时间
├── context TEXT           -- 提问时的业务上下文
└── session_id TEXT        -- 所在会话

knowledge_points（知识点库）
├── id TEXT PK
├── category TEXT          -- 分类：accounting/tax/law/practice
├── title TEXT             -- 知识点标题
├── content TEXT           -- 知识点内容（Markdown）
├── difficulty TEXT        -- 难度：basic/intermediate/advanced
├── related_points TEXT    -- 关联知识点（JSON数组）
└── source TEXT            -- 来源（准则条文编号等）

user_mastery（用户掌握度）
├── user_id TEXT
├── knowledge_point_id TEXT
├── status TEXT            -- unlearned/aware/mastered
├── first_seen_at REAL
├── last_reviewed_at REAL
├── review_count INTEGER
└── PRIMARY KEY (user_id, knowledge_point_id)
```
