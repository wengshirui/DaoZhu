# 049 — 工具调用可视化（指挥官体验）

**Intake ID:** 2026-05-28-tool-call-visual
**Requester:** PO
**Urgency:** Normal
**Status:** 待澄清

---

## 原始需求

> 现在对话特别是调用工具不够酷，用户没有一种指挥一切的感觉。把工具调用弹窗打开，然后滚动显示工作过程。

## 提取意图

当前工具调用只是一行简单的状态文字（"🔧 list_workspaces → ✅ 完成"），缺乏沉浸感。用户希望有一种"指挥官下达命令，看着系统执行"的体验。

改进方向：工具调用时弹出一个可视化面板/终端窗口，实时滚动显示执行过程（工具名、参数、返回结果摘要），让用户感受到 AI 在"干活"。

---

## 开放问题

| # | 模块 | 问题 | AI 猜测 | 答案 |
|---|------|------|---------|------|
| 1 | 视觉形式 | 工具执行过程的展示形式是什么样的？终端风格滚动日志、卡片式步骤流、还是侧边面板？ | 终端风格（黑底绿字/暗色面板），在聊天区内展开 | |
| 2 | 信息粒度 | 用户需要看到多少细节？只看工具名+状态，还是包括参数和返回结果？ | 工具名 + 简要参数 + 结果摘要（不超过 2 行） | |
| 3 | 交互 | 执行完成后面板是自动收起还是保持展开？用户能否手动折叠？ | 执行完自动收起为一行摘要，可点击展开查看详情 | |
