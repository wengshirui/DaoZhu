# REQ-012 记忆与技能系统

**状态：** ✔️ Done  
**优先级：** P1（提升智能化体验的关键）  
**T-Shirt Size：** M - 参考 Hermes 已有实现，适配财务领域

---

## 问题陈述

财务工作有大量重复模式：同一类业务总是做相同的分录，同一个客户的发票格式总是一样的。AccoBot 需要能"记住"用户的偏好和习惯，越用越聪明。同时，财务规则（如报销标准、税率政策）需要以结构化的方式存储，让 AI 在处理业务时能准确引用。

## 用户故事

**As a** 用户  
**I want** AI 记住我的做账习惯，下次遇到类似业务直接按我的方式处理  
**So that** 我不用每次都纠正 AI 的建议，效率越来越高

## 范围

### In Scope

**记忆系统：**
- **分录模式记忆** — 记住用户修正过的科目匹配（如："快递费"→ 管理费用-邮寄费）
- **偏好记忆** — 用户的常用科目、默认摘要模板、习惯的凭证分类方式
- **上下文记忆** — 跨会话记住用户的公司信息、行业特征、常见业务
- **记忆衰减** — 长期不使用的记忆降低权重，避免过时信息干扰

**技能系统：**
- **财务规则 Skill** — 以 Markdown 文件存储的财务知识（如：差旅费报销标准、各税种申报流程）
- **行业模板 Skill** — 特定行业的常见业务分录模板（如：餐饮业、电商、建筑业）
- **操作流程 Skill** — Playwright 自动化的操作步骤记录
- **Skill 可编辑** — 用户可查看和修改 Skill 内容
- **Skill 可分享** — 导出/导入 Skill 文件

### Out of Scope

- AI 自动创建 Skill（先做手动创建 + AI 辅助编辑）
- Skill 市场/社区分享平台

## 验收标准

1. **AC-1：分录学习** — 用户将"快递费"的科目从"办公费"改为"邮寄费"后，下次遇到"快递费"直接建议"邮寄费"
2. **AC-2：模式识别** — 连续 3 次相同模式的业务后，Agent 主动提出"要不要我记住这个做法"
3. **AC-3：Skill 加载** — 用户说"按差旅费报销标准处理"，Agent 加载对应 Skill 并按规则执行
4. **AC-4：Skill 编辑** — 用户说"把差旅费标准里的住宿上限改为500元"，Agent 更新 Skill 文件
5. **AC-5：记忆查看** — 用户说"你记住了哪些关于我的信息"，Agent 列出当前记忆内容
6. **AC-6：记忆清除** — 用户说"忘掉关于XX的记忆"，Agent 删除指定记忆

## 数据存储

- 记忆存储在 `chat.db` 的专用表中（key-value + 向量检索）
- Skill 文件存储在 `~/.accobot/skills/` 目录下（Markdown 格式）
- 分录模式存储在 `accounting.db` 的模式表中
