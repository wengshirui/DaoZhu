# 043 — Gitee 远程控制与聊天

> 状态: 🆕 待开发 | Size: M (5 pts) | 优先级: P0

---

## 问题陈述

用户目前只能在本地浏览器（localhost:7788）访问岛主，无法从手机、公司电脑等其他设备远程操控。需要一种：
- 无需公网 IP / 内网穿透
- 无需注册第三方聊天平台
- 用户已有 Gitee 账号即可使用

的远程交互方式。

---

## 方案

利用 Gitee Issues 作为双向消息通道：

```
用户（手机/其他设备）          岛主（本地运行）
       │                           │
       │  1. 发 Issue / 评论        │
       ├──────────────────────────▶│
       │                           │  2. 轮询 / Webhook 接收
       │                           │  3. Agent 处理指令
       │  4. Issue 评论回复         │
       │◀──────────────────────────┤
       │                           │
```

## User Stories

### Story 1: 远程发送指令

> As a 岛主用户，I want 在 Gitee 仓库发一条 Issue 评论作为指令，so that 我在任何设备上都能操控我的数字小岛。

**验收标准：**
1. 用户在指定仓库的"远程控制"Issue 下发评论，岛主在 60s 内识别
2. 支持的指令：查看工作区列表、启停工作区、简单对话
3. Agent 处理结果以评论形式回复，格式清晰可读
4. 非法指令返回友好错误提示（不暴露内部错误）

### Story 2: 轮询服务

> As a 岛主平台，I want 后台定时轮询 Gitee Issue 评论，so that 无需公网 IP 也能接收远程指令。

**验收标准：**
1. 轮询间隔可配置（默认 30s），空闲时 CPU < 1%
2. 仅处理未回复的新评论（基于时间戳去重）
3. Gitee API 限流时自动退避（指数退避，最大 5min）
4. 轮询服务异常不影响主平台运行（独立线程/协程）

### Story 3: 安全与权限

> As a 岛主用户，I want 只有我自己的 Gitee 账号能控制我的岛，so that 别人无法通过 Issue 操控我的系统。

**验收标准：**
1. 仅响应配置中指定的 Gitee 用户名发出的评论
2. 危险操作（删除类）在远程模式下拒绝执行，提示"请在本地操作"
3. 配置页面可开关远程控制功能（默认关闭）

## 技术设计要点

### 消息通道选择

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| Gitee Issue 评论 | 无需公网IP、天然持久化、手机可用 | 延迟 30-60s、API 限流 | ✅ 主方案 |
| Gitee Webhook | 实时、低延迟 | 需公网IP或内网穿透 | 备选（进阶） |
| Gitee WebSocket | 实时 | Gitee 不提供 | ❌ |

### 模块划分

```
daozhu/
├── services/
│   └── remote_control.py    # 轮询服务 + 指令解析 + 回复
├── routes/
│   └── remote.py            # 远程控制配置 API
```

### 指令格式

用户在 Issue 评论中直接写自然语言，Agent 解析意图：

```
用户评论: 启动财务助手
Agent回复: ✅ 财务助手已启动（端口 7803）
          当前运行中的工作区：个人待办、财务助手
```

### 配置项

```json
{
  "remote_control": {
    "enabled": false,
    "gitee_repo": "yumen2278/DaoZhu",
    "issue_number": null,
    "allowed_users": ["yumen2278"],
    "poll_interval_seconds": 30
  }
}
```

## 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| Gitee API 限流（5000次/小时） | 轮询中断 | 指数退避 + 缓存最后处理时间 |
| 评论顺序错乱 | 指令重复执行 | 基于评论 ID 去重 |
| 用户误操作 | 远程删除数据 | 远程模式禁止危险操作 |
