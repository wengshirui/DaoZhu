# REQ-013 消息平台接入

**状态：** ✔️ Done  
**优先级：** P2（第四阶段）  
**T-Shirt Size：** M - 参考 Hermes gateway 模式，每个平台一个适配器

---

## 问题陈述

财务人员不一定总在电脑前。通过微信/钉钉/飞书等日常通讯工具与 AccoBot 交互，可以随时随地查账、记账、收到报税提醒。

## 用户故事

**As a** 用户  
**I want** 在微信里直接问 AI "这个月还剩多少钱"  
**So that** 不用打开电脑也能随时了解财务状况

## 范围

### In Scope

- **微信** — 通过企业微信或个人微信机器人接入
- **钉钉** — 钉钉机器人
- **飞书** — 飞书机器人
- **统一网关** — 所有平台共用一个消息处理引擎，每个平台只需实现收发消息的适配器
- **消息类型** — 文本、图片（发票拍照）、文件（Excel 流水）
- **主动推送** — 报税提醒、异常预警等主动通知

### Out of Scope

- 语音消息识别
- 群聊场景（先做 1v1 私聊）
- 审批流程集成

## 验收标准

1. **AC-1：文本对话** — 用户在微信/钉钉/飞书中发送文本消息，Agent 能正确响应
2. **AC-2：图片处理** — 用户发送发票照片，Agent 能接收并进行 OCR 识别
3. **AC-3：文件处理** — 用户发送 Excel 文件，Agent 能接收并处理（如银行流水导入）
4. **AC-4：主动推送** — 报税截止日期前，Agent 主动向用户发送提醒消息
5. **AC-5：平台隔离** — 不同平台的消息互不干扰，但共享同一个账套数据
6. **AC-6：新平台扩展** — 新增一个消息平台只需实现 `MessageAdapter` 接口（收消息/发消息/发文件），无需修改核心逻辑
7. **AC-7：界面化配置** — Gateway 的平台配置（企业微信 CorpID/Secret、钉钉 AppKey、飞书 AppID 等）通过 Web UI 设置页面完成，与 API Key 配置体验一致，不要求用户编辑 .env 文件

## 架构设计

```
用户消息 → 平台适配器（微信/钉钉/飞书）→ 统一消息网关 → Agent 核心 → 响应
                                                              ↓
                                                         工具调用
                                                              ↓
                                                         返回结果
```
