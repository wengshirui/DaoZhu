# REQ-011 浏览器自动化（Playwright）

**状态：** ✔️ Done  
**优先级：** P0（财务核心能力 — 与税务局/工商/社保等系统交互的唯一途径）  
**T-Shirt Size：** L - 工具层已完成，MCP 集成为默认配置

---

## 问题陈述

财务人员日常需要登录多个 B/S 架构的外部系统完成重复性操作：电子税务局报税、网银下载回单、开具发票等。这些操作步骤固定但耗时，且各系统没有开放 API。Playwright 浏览器自动化可以模拟人工操作，在用户已登录的会话中代替用户完成这些重复工作。

## 用户故事

**As a** 用户  
**I want** 登录好税务系统后，告诉 AI "帮我把这个月的增值税申报表提交"  
**So that** AI 自动在税务系统里填写数据并提交，我不用手动一个个填

## 范围

### In Scope

- **电子税务局** — 增值税申报、企业所得税申报、附加税申报
- **网银系统** — 下载银行回单、导出对账单
- **发票平台** — 开具发票、查验发票真伪
- **社保公积金** — 查询缴费明细、下载对账单
- **财务软件 Web 版** — 金蝶云/用友云等的数据导入导出
- **操作录制** — 用户可录制自定义操作流程，AI 学习后自动重放

### Out of Scope

- 自动登录（安全红线：不存储任何外部系统的账号密码）
- C/S 架构的桌面软件自动化
- 绕过验证码（需要用户手动处理验证码环节）

## 安全原则

1. **不存储凭据** — 项目不记录、不缓存任何外部系统的用户名和密码
2. **用户授权登录** — 用户自行在浏览器中完成登录，Bot 在已认证的会话中操作
3. **操作确认** — 涉及提交/付款等不可逆操作前，必须向用户确认
4. **操作日志** — 所有自动化操作记录完整日志，可追溯审计
5. **随时中断** — 用户可随时中断自动化操作

## 验收标准

1. **AC-1：会话接管** — 用户在浏览器中登录外部系统后，Agent 能接管该浏览器会话进行操作
2. **AC-2：数据填写** — Agent 能将 AccoBot 中计算好的申报数据自动填入外部系统的表单
3. **AC-3：提交确认** — 在点击"提交"按钮前，Agent 暂停并向用户展示即将提交的内容，等待确认
4. **AC-4：错误处理** — 外部系统页面结构变化导致操作失败时，Agent 能识别异常并通知用户，不会盲目继续
5. **AC-5：操作录制** — 用户可以录制一段操作流程（如：登录某系统→点击某菜单→下载文件），Agent 学习后可重放
6. **AC-6：默认安装** — ✔️ Playwright 作为项目默认依赖安装（不是可选），因为与政府/银行系统交互是财务工作的刚需
7. **AC-7：MCP 集成** — ✔️ Playwright 通过 MCP（Model Context Protocol）方式集成，作为默认 MCP Server 配置，无需用户手动配置
8. **AC-8：日志审计** — 每次自动化操作生成完整日志（时间、操作步骤、截图），用户可回溯查看
