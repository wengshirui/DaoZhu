# REQ-032: 三层进程模型（轻量/标准/重型工作区）

> Intake ID: 2026-05-26-process-model
> 优先级: 🔴 高（架构优化，解决资源浪费）

---

## 问题陈述

当前每个工作区 = 独立进程 + 独立端口。10 个工作区 = 10 个 Python 进程，内存开销大。简单的待办、笔记不值得独立跑一个 uvicorn。

## 三层模型

| 层级 | 模式 | 适用场景 | venv |
|------|------|----------|------|
| L1 轻量 | 挂载到主进程子路由 | 简单 CRUD（≤3 表，无额外依赖） | 共享主环境 |
| L2 标准 | 独立进程 + 独立端口 | 复杂业务、自定义依赖 | 独立 venv |
| L3 重型 | 独立进程 + 外部服务 | Jupyter/Grafana/第三方集成 | 独立 venv + 系统依赖 |

## 判断规则

- schema.sql ≤ 3 表 + 无额外依赖 → L1
- schema.sql > 3 表 或 有额外依赖 → L2
- 集成外部服务 → L3

## workspace.json 新增字段

```json
{ "mode": "lightweight" | "standard" | "heavy" }
```

## 验收标准

| # | AC | 可测试性 |
|---|-----|----------|
| 1 | mode=lightweight 的工作区挂载到主进程，不占独立端口 | 启动后检查进程数 |
| 2 | mode=standard 的工作区仍独立进程运行 | 同之前行为 |
| 3 | 创建工作区时 Agent 自动判断 mode | 创建简单工作区验证 |
| 4 | 轻量工作区仍有独立 data.db | 检查数据隔离 |
