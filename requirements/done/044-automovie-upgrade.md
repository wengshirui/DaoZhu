# 044 — AutoMovie 方案升级

## 问题陈述

当前 AutoMovie（话剧导演）是纯 HTML/Canvas 实现。需要评估是否升级为 Godot 项目，或继续深化 JS 方案。核心诉求：
- 客户端完全自主（不依赖浏览器）
- 离线可用
- 可导出为独立可分发文件
- 表现力足够丰富

---

## ⚠️ 待讨论：Godot vs JS 方案对比

> 此需求需要先完成技术方案讨论，再决定实施路径。

### 方案 A: 继续深化 JS（HTML/Canvas/WebGL）

**做法：** 在现有 demo.html 基础上迭代，引入更完善的实体系统、场景编辑器、时间轴。

**优点：**
- 零学习成本，团队已有 JS 经验
- 与岛主平台天然集成（同为 Web 技术栈）
- 轻量工作区模式可直接挂载到主进程
- 分发简单：一个 HTML 文件发给任何人，浏览器打开即用
- 开发速度快，迭代周期短
- 可用 Electron/Tauri 打包为桌面应用（如果需要独立客户端）
- 生态丰富：PixiJS / Phaser / Three.js 等成熟库可选

**缺点：**
- Canvas 2D 表现力有天花板（复杂粒子、光影、骨骼动画较难）
- 浏览器沙箱限制（文件系统访问、本地资源管理）
- 导出视频需要额外方案（canvas-to-video 库或服务端录制）
- 性能在复杂场景下可能不如原生

### 方案 B: 迁移到 Godot 4.x

**做法：** 用 Godot 重写客户端，AI 部分通过 HTTP API 驱动 Godot。

**优点：**
- 原生性能，2D/3D 表现力强
- 内置动画系统、粒子系统、音频系统
- 一键导出多平台（Windows/Mac/Linux/Android/Web）
- 内置录制/截图能力
- 独立 exe，不依赖浏览器
- GDScript 学习曲线平缓（类 Python 语法）
- 完全开源免费，无授权费

**缺点：**
- 新技术栈引入，团队需要学习 GDScript + Godot 编辑器
- 与岛主平台集成需要额外通信层（HTTP/WebSocket）
- 项目体积增大（Godot 运行时 ~30-50MB）
- 迭代速度初期较慢
- 用户需要额外下载 Godot 客户端（或打包后的 exe）
- 调试链路变长：Python Agent → HTTP → Godot

### 方案 C: 混合方案（JS 为主 + Godot 可选导出）

**做法：** 日常开发和预览用 JS/Canvas（Web），需要高质量导出时调用 Godot 做渲染。

**优点：**
- 兼顾开发效率和最终品质
- 用户日常使用零门槛（浏览器即可）
- 需要"电影级"输出时才启动 Godot 渲染管线

**缺点：**
- 维护两套渲染逻辑，长期成本高
- 剧本格式需要同时兼容两端
- 复杂度翻倍

### 对比总结

| 维度 | JS 方案 | Godot 方案 | 混合方案 |
|------|---------|------------|----------|
| 学习成本 | 零 | 中等 | 中等 |
| 开发速度 | 快 | 初期慢 | 中等 |
| 与岛主集成 | 天然 | 需通信层 | 部分天然 |
| 表现力 | 中等 | 强 | 强（导出时） |
| 分发便利性 | 极高（HTML文件） | 高（exe） | 高 |
| 离线能力 | 部分 | 完全 | 完全（Godot端） |
| 视频导出 | 需额外方案 | 内置 | 内置（Godot端） |
| 长期维护 | 低 | 中等 | 高 |
| 用户门槛 | 最低 | 需下载客户端 | 最低（日常） |

### 决策建议

**短期（1-2 周）：** 继续深化 JS 方案，把 demo.html 做成可用的工作区，验证"AI 导演"核心玩法。

**中期决策点：** 当 JS 方案遇到以下瓶颈时，启动 Godot 迁移：
- 用户明确需要视频导出功能
- 场景复杂度超出 Canvas 2D 能力
- 需要移动端原生体验

**建议先做 Spike：** 花 1-2 天验证 Godot ↔ Python HTTP 通信可行性，评估实际集成难度。

## User Story（无论哪个方案都适用）

> As a 创作者，I want 用自然语言描述剧情后看到实体按剧本表演，so that 我能快速将想法变成可视化动画。

**验收标准：**
1. 输入一段自然语言，系统生成 JSON 剧本并执行
2. 支持至少 5 种实体类型和 10 种动作
3. 支持重放、暂停、步进调试
4. 演出结果可导出为可分享的文件（HTML 或 exe 或视频）
5. AI 导演接口与 AGENT.md 中定义的 JSON 格式兼容

---

## 技术 Spike 任务（优先执行）

1. **JS 方向：** 将 demo.html 改造为岛主工作区（workspace.json + 轻量挂载），验证 Agent 调用链路
2. **Godot 方向：** 创建最小 Godot 项目，实现 HTTP Server 接收 JSON 剧本 → 移动一个 Sprite，验证通信可行性
3. **对比报告：** 两个 Spike 完成后输出对比文档，做最终技术选型
