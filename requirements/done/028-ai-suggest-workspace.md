# REQ-028: AI 根据记忆自动优化资源

> Intake ID: 2026-05-26-ai-auto-optimize
> 优先级: 🟡 中（越用越好的核心）

---

## 问题陈述

岛管理员有记忆系统，知道用户偏好和使用习惯。应该能主动建议优化——不仅是工作区，还包括 Skill 和工具（MCP），让整个岛屿越用越适合用户。

## 用户故事

**As a** 岛主用户，
**I want** 岛管理员根据我的使用记忆主动建议新增/优化资源，
**so that** 我的岛屿越来越符合我的需求，不需要我主动想到每个优化点。

## 建议范围

| 类型 | 触发条件 | 建议示例 |
|------|----------|----------|
| 新建工作区 | 多次提到某类需求 | "你经常提到读书，要不要建一个读书笔记工作区？" |
| 优化工作区 | 使用中发现不足 | "你的待办工作区可以加个标签筛选功能" |
| 新增 Skill | 重复执行某类操作 | "你经常让我搜索开源项目，我可以把这个流程固化为一个技能" |
| 优化 Skill | Skill 成功率低 | "create-workspaces 技能最近失败率较高，我来优化一下" |
| 禁用工具 | 工具长期未使用 | "list_templates 工具你从没用过，要不要禁用它节省资源？" |

## 实现方式

在 Agent 的 system prompt 中注入建议逻辑，基于：
1. `user_profile` — 用户偏好
2. `knowledge` — 历史经验
3. `skill_usage` — 工具/技能使用统计
4. `stale skills` — 长期未使用的资源

## 验收标准

| # | AC | 可测试性 |
|---|-----|----------|
| 1 | 对话中多次提到某类需求时，管理员主动建议创建工作区 | 多次提到"读书"后验证 |
| 2 | 建议以自然对话形式呈现，不弹窗不打断 | 对话中出现建议 |
| 3 | 用户同意后管理员调用工具执行 | 同意后检查结果 |
| 4 | 管理员能建议禁用长期未用的工具 | 30天未用后验证 |
| 5 | 管理员能建议优化失败率高的 Skill | 模拟失败后验证 |

## T-Shirt Size

**M (5 pts)** — system prompt 增强 + 记忆数据注入 + 建议触发逻辑
