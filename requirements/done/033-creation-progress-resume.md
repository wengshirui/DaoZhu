# REQ-033: 工作区创建进度记录 + 断点续做

> Intake ID: 2026-05-26-creation-progress
> 优先级: 🟡 中

---

## 问题陈述

工作区创建过程可能中断（网络断、关闭浏览器）。不用传统回滚，改为记录进度，下次进来断点续做。

## 用户故事

**As a** 岛主用户，
**I want** 工作区创建中断后下次能继续，
**so that** 不会因为意外中断而丢失已完成的步骤。

## 验收标准

| # | AC | 可测试性 |
|---|-----|----------|
| 1 | 创建工作区前在 SQLite 记录步骤清单和当前进度 | 检查数据库 |
| 2 | 中断后重新进入，提示"上次创建未完成，是否继续？" | 模拟中断验证 |
| 3 | 续做时从上次完成的步骤继续，不重复执行 | 验证步骤不重复 |

## T-Shirt Size

**S (3 pts)** — SQLite 进度表 + Agent 检查逻辑
