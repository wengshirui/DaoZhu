# REQ-018: 好玩 — 发现的快乐

> Intake ID: 2025-05-25-fun-discovery
> 优先级: 🟡 中（差异化核心）

---

## 核心理念

> 岛是一个放松度假的地方。不是打卡、不是任务、不是等级压力。
> 是那种——"咦，这里还有这个？"的发现的快乐。

**不要**：任务系统、等级、经验值、连续打卡、排行榜
**要**：彩蛋、惊喜、小细节、会心一笑、探索奖励

---

## 用户故事

**As a** 岛主用户，
**I want** 在使用过程中不经意间发现有趣的小细节和彩蛋，
**so that** 我觉得这个产品有温度、有灵魂，愿意继续探索。

---

## 好玩的方向

### 1. 岛管理员的小动作（已有基础）

- 空闲时偶尔打哈欠、伸懒腰
- 用户长时间不操作时，管理员坐下来看书
- 深夜使用时管理员打着手电筒
- 节日时管理员换装（春节穿红、圣诞戴帽）

### 2. 岛屿天气/氛围

- 根据真实时间变化：早晨暖光、傍晚橙色、深夜星空
- 偶尔下雨（背景有雨滴动画）
- 用户创建新工作区时放烟花

### 3. 隐藏彩蛋

- 连续点击标题 5 次 → 管理员跳舞
- 输入特定关键词（如"你好世界"）→ 彩蛋回复
- 第一次创建工作区 → 管理员鼓掌 + 撒花动画
- 发现所有工作区的 README → "探索家"徽章（不是任务，是自然发现）

### 4. 随机小惊喜

- 管理员偶尔在对话中加入 emoji 表情或俏皮话
- 状态栏偶尔显示有趣的提示（"今天天气不错，适合建造"）
- 工作区卡片 hover 时偶尔有不同的微动画

### 5. 岛屿成长（轻量版，非等级）

- 不是等级系统，是**岛屿变化**
- 工作区越多 → 书架越满（视觉变化）
- bug越多→ 岛上螃蟹越多
- 用户情绪会改变岛上天气
- 使用时间越长 → 岛上多了些装饰（花草、小动物）
- 纯视觉变化，没有数字、没有进度条、没有压力

## 验收标准（MVP — 先做最轻量的）

| # | AC | 可测试性 |
|---|-----|----------|
| 1 | 根据当前时间（早/午/晚/夜），界面背景色微调 | 不同时段打开验证 |
| 2 | 用户创建第一个工作区时，有撒花/庆祝动画 | 创建后观察 |
| 3 | 连续点击顶栏标题 5 次，管理员跳舞 | 点击验证 |

## 技术方案

引入 **KAPLAY**（原 Kaboom.js 继承者，⭐4k）实现星露谷风格像素岛屿场景：

```html
<script src="https://unpkg.com/kaplay@3001/dist/kaplay.js"></script>
```

- 单文件 CDN 引入，零构建工具
- API 极简：`kaplay(); loadSprite("player", "sprite.png"); add([sprite("player"), pos(100,100)])`
- 星露谷风格像素游戏天然适配
- 管理员在像素岛上走动、与工作区建筑互动
- 用 [Universal LPC Spritesheet Generator](https://liberatedpixelcup.github.io/Universal-LPC-Spritesheet-Character-Generator/) 生成角色 sprite sheet

## 设计原则

- **轻量** — 不增加用户认知负担
- **自然** — 不弹窗、不通知、不打断
- **发现** — 用户自己发现才有趣，不要主动告知
- **温暖** — 像度假岛上的小惊喜，不是游戏任务
