# REQ-001: 岛主主界面前端

> Intake ID: 2025-05-25-main-ui-frontend

---

## 原始需求

构建岛主平台的主界面。三栏布局：左侧导航面板（建筑/技能/工具）、中间聊天窗口、右侧信息面板（历史对话/输出日志）。默认提供三个参考工作区：个人待办、岛主论坛、个人记账。

---

## 提炼后的需求

### 问题陈述

用户需要一个功能完整的主界面来与平台交互。界面需要同时承载：资源管理（工作区、技能、工具）、AI 对话、历史记录查看等核心功能。当前没有任何可视化入口。

### 范围

**包含（In Scope）：**

- 三栏布局主界面（左侧导航 + 中间聊天 + 右侧信息）
- 左侧面板：建筑（工作区列表）、技能（Skill 列表）、工具（MCP 列表）三个 Tab
- 中间区域：AI 聊天窗口（消息列表 + 输入框）
- 右侧面板：历史对话列表、输出日志两个 Tab
- 三个默认工作区卡片：个人待办、岛主论坛、个人记账
- 响应式布局（移动端可折叠侧栏）
- 顶部标题栏

**不包含（Out of Scope）：**

- AI 对话的后端实现（仅前端 UI 壳）
- 工作区实际运行/进程管理
- 市场安装功能
- 技能/工具的实际调用逻辑

### 用户故事

**As a** 岛主用户，
**I want** 打开浏览器看到一个三栏布局的工作台界面，左侧管理资源、中间与管家对话、右侧查看历史和日志，
**so that** 我能在一个界面内完成所有操作，不需要在多个页面间切换。

### 验收标准

| # | 验收标准 | 可测试性 |
|---|----------|----------|
| AC-1 | 访问 `http://localhost:7788` 返回三栏布局页面，左/中/右区域均可见 | DOM 结构检查 |
| AC-2 | 左侧面板包含"建筑""技能""工具"三个 Tab，点击切换显示对应列表 | 点击 Tab 验证内容切换 |
| AC-3 | "建筑" Tab 默认展示 3 个工作区卡片：个人待办(📋)、岛主论坛(🏝️)、个人记账(💰)，每张卡片显示图标、名称、状态 | 视觉验证 + DOM 检查 |
| AC-4 | 中间聊天区域包含消息展示区和底部输入框，输入框支持回车发送（前端仅展示，无需后端响应） | 输入文字回车后消息出现在列表 |
| AC-5 | 右侧面板包含"历史对话""输出日志"两个 Tab，点击切换显示对应内容 | 点击 Tab 验证内容切换 |
| AC-6 | 点击运行中的工作区卡片，在新标签页打开该工作区页面 | 检查 window.open 调用 |
| AC-7 | 页面在 768px 以下宽度时，左右侧栏自动折叠为可展开的抽屉 | 缩小视口验证 |
| AC-8 | 页面加载失败或 API 请求失败时，显示友好错误提示 | 模拟网络错误验证 |

### 业务价值

- **产品核心体验**：主界面是用户与平台的唯一交互入口，决定产品可用性
- **MVP 交付基础**：没有主界面，平台无法作为产品交付
- **量化指标**：用户可在 5 秒内找到目标工作区并打开，或开始与管家对话

### T-Shirt Size

**M (5 pts)** — 三栏布局 + 多 Tab 切换 + 聊天 UI + 响应式适配，前端复杂度中等；依赖后端提供工作区/技能/工具列表 API；无数据库变更。

### 依赖与约束

| 依赖 | 说明 |
|------|------|
| 后端 API | `GET /api/workspaces` 返回工作区列表 |
| 后端 API | `GET /api/skills` 返回技能列表 |
| 后端 API | `GET /api/tools` 返回工具列表 |
| 后端 API | `GET /api/conversations` 返回历史对话列表 |
| workspace.json | 工作区元信息格式已定义 |

| 约束 | 说明 |
|------|------|
| 无 Node.js | 纯 HTML + CSS + JS |
| 文件 ≤ 500 行 | 遵循 AGENTS.md 文件规模约束 |
| 无框架 | 不使用 React/Vue |
| 可访问性 | 语义化 HTML，键盘可导航 |

## 实现任务拆分

### Story 1: 三栏布局骨架 (2 pts)

**As a** 用户，**I want** 看到三栏布局的页面骨架，**so that** 各功能区域清晰分隔。

| Task | 描述 | 预估 |
|------|------|------|
| 1.1 | 创建 index.html（三栏结构 + 顶部栏 + 状态栏） | 0.5d |
| 1.2 | 编写 base.css（CSS 变量 + 三栏 flexbox 布局 + 响应式断点） | 0.5d |
| 1.3 | 编写 layout.css（面板折叠/展开动画 + 移动端抽屉） | 0.5d |

### Story 2: 左侧资源面板 (2 pts)

**As a** 用户，**I want** 在左侧面板切换查看工作区、技能、工具列表，**so that** 我能管理平台上的所有资源。

| Task | 描述 | 预估 |
|------|------|------|
| 2.1 | 编写 api.js（fetch 封装 + 错误处理） | 0.5d |
| 2.2 | 编写 sidebar.js（Tab 切换 + 列表渲染 + 工作区卡片点击） | 1d |
| 2.3 | 编写 components.css（卡片样式 + Tab 样式 + 状态徽章） | 0.5d |

### Story 3: 中间聊天窗口 (2 pts)

**As a** 用户，**I want** 在中间区域与管家对话，**so that** 我能通过自然语言指挥管家工作。

| Task | 描述 | 预估 |
|------|------|------|
| 3.1 | 编写 chat.js（消息渲染 + 输入发送 + 自动滚动） | 1d |
| 3.2 | 编写 chat.css（消息气泡 + 输入框 + 打字指示器） | 0.5d |

### Story 4: 右侧信息面板 (1 pt)

**As a** 用户，**I want** 在右侧查看历史对话和输出日志，**so that** 我能回顾之前的操作和排查问题。

| Task | 描述 | 预估 |
|------|------|------|
| 4.1 | 编写 panel.js（历史对话列表 + 日志 Tab 切换 + 日志流式显示） | 1d |
| 4.2 | 编写 panel.css（列表样式 + 日志样式） | 0.5d |

### Story 5: 整合与容错 (1 pt)

| Task | 描述 | 预估 |
|------|------|------|
| 5.1 | 编写 app.js（初始化流程 + 全局错误处理 + 主题切换） | 0.5d |
| 5.2 | 响应式适配测试 + 可访问性检查 | 0.5d |
