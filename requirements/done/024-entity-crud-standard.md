# REQ-024: 工作区代码规范 — 实体 CRUD 标准

> Intake ID: 2025-05-25-entity-crud-standard
> 优先级: 🟡 中（规范类，影响后续所有工作区质量）

---

## 问题陈述

工作区代码生成时，每个实体的 API 应该有统一标准。除非特殊要求，每个实体最起码要有增删改查，以及对关联实体的影响处理。

## 规范内容

每个实体（数据库表）必须提供：

### 必须的 API

| 操作 | 路由 | 说明 |
|------|------|------|
| 列表 | GET /api/{entity}/ | 支持分页、筛选、排序 |
| 详情 | GET /api/{entity}/{id} | 含关联数据 |
| 创建 | POST /api/{entity}/ | 含数据校验 |
| 更新 | PUT /api/{entity}/{id} | 部分更新 |
| 删除 | DELETE /api/{entity}/{id} | 含关联影响处理 |

### 关联影响规则

| 场景 | 处理方式 |
|------|----------|
| 删除父实体 | 子实体级联删除 OR 置空外键（按业务决定） |
| 删除被引用实体 | 拒绝删除并提示"该 XX 被 N 条 YY 引用" |
| 状态变更 | 通知关联实体（如凭证过账后锁定分录） |

### 必须的字段

每个实体表必须包含：
- `id` — 主键
- `created_at` — 创建时间
- `updated_at` — 更新时间

## 用户故事

**As a** 使用 create-workspaces 技能的管家，
**I want** 生成的工作区代码遵循统一的 CRUD 标准，
**so that** 每个工作区的 API 质量一致，用户体验统一。
