# REQ-021 凭证事后质检自动触发

**状态：** 📝 Draft  
**优先级：** S  
**来源：** 2025-05-21 架构 Grill 讨论

---

## 问题陈述

当前质检工具（quality_check_tool）需要 Agent 主动调用或用户手动触发。对于不懂会计的用户，他们无法判断 Agent 生成的分录是否正确。需要在凭证生成后自动触发质检，作为过账前的安全关卡，确保基本的会计规则不被违反。

## 范围

**In Scope:**
- 凭证创建/修改后自动触发质检（不需要用户或 Agent 主动调用）
- 质检结果附加到凭证记录上（通过/不通过 + 具体问题）
- 质检不通过时阻止过账，并向用户解释问题
- 基于会计恒等式（资产+费用=负债+所有者权益+收入）的约束检查

**Out of Scope:**
- 科目选择正确性的深度判断（依赖模型推理，非规则可覆盖）
- 跨期间的连续性检查
- 与外部审计系统对接

## 用户故事

As a 不懂会计的小微企业老板，I want 系统自动检查 Agent 做的账对不对，so that 我不会因为 AI 犯错而产生错误的财务数据。

## 验收标准

1. **AC-1 自动触发**：凭证创建成功后，系统自动执行质检，无需用户或 Agent 额外调用
2. **AC-2 规则检查**：至少包含以下检查项——借贷平衡、科目余额方向异常（资产类出现贷方余额等）、大额现金交易（>5万）、空摘要、会计恒等式验证
3. **AC-3 结果记录**：质检结果（通过/不通过 + 问题列表）持久化存储，关联到对应凭证
4. **AC-4 过账拦截**：质检不通过的凭证，过账操作返回错误并说明具体问题，要求修正后重新质检
5. **AC-5 用户可读**：质检问题描述使用通俗语言（如"这笔分录借方和贷方金额不相等"而非"借贷不平衡"）

## 业务价值

- 防止错误数据入账：一旦过账后修正成本高（需红冲+重做）
- 建立用户信任：即使不懂会计，也知道系统有"检查员"在把关
- 合规基础：为未来对接税务审计提供数据质量保障

## T-Shirt Size

**S** - 质检逻辑已有（REQ-015），主要工作是改造触发时机（从手动到自动）+ 结果持久化 + 过账拦截逻辑；低风险

## 依赖与约束

- 依赖 REQ-015（现有质检规则）
- 依赖 REQ-003/004（凭证创建和过账流程）
- 约束：质检不应显著增加凭证创建的响应时间（<500ms）
