## 提取意图（拆分为 4 个独立需求）

### 需求 A：Agent 分层解决策略

Agent 在处理用户问题时，采用**三级递进**机制：
1. **Level 1 — 本地优先**：用工作区数据、已有技能、本地工具解决
2. **Level 2 — 联网兜底**：本地搞不定时，经用户同意后用 Playwright 联网搜索/操作，限制上下文长度和工具调用次数
3. **Level 3 — 无限循环**：联网也搞不定时，经用户同意后进入"死磕模式"，不限次数直到解决，但需提示 token 消耗警告

### 需求 B：待办今日聚焦 → 桌面侧边栏

将"个人待办"工作区的"今日聚焦"功能做成桌面侧边栏形式，常驻显示当日任务。

### 需求 C：Agent 配置增加 Python 兜底策略

在 Agent 的工作要求/配置中增加一条规则：如果简单回答和联网都无法解决用户问题，尝试用 Python 脚本来解决（写代码、跑脚本、分析数据等）。

### 需求 D：桌面宠物工作区（参考 codex-pet）

新建一个"桌面宠物"工作区，参考 [codex-pet.org](https://codex-pet.org/zh/#gallery) 的宠物合集，实现：
1. 从开源官网选择/下载宠物
2. 管理宠物、互动（喂食、喂水、打扮等）
3. 宠物可以"跑出"工作区，在岛主主界面、其他工作区、甚至电脑桌面上活动
