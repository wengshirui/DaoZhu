## 需求管理规范

### 文件结构

```
requirements/
├── plan.md         # 本文件 — 需求总览
├── backlog/        # 待开发需求（详细文档）
└── done/           # 已完成/已取消需求（归档）
```

### 状态

| 图标 | 含义 |
|------|------|
| ✅ | 已完成（归档到 done/） |
| 🚧 | 进行中 |
| 🆕 | 待开发（详见 backlog/） |
| ❌ | 已取消 |

### 需求文件命名

`{三位数ID}-{kebab-case-slug}.md`，如 `052-desktop-pet-workspace.md`
