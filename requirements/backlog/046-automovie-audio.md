## 原始需求

> 火柴人剧场要增加背景音乐和配音

## 提取意图

AutoMovie（火柴人话剧导演）工作区目前只有视觉动画输出，缺少音频维度。需要为生成的剧场动画增加：
1. 背景音乐（BGM）— 烘托氛围
2. 角色配音（TTS）— 让角色"说话"

---

## 开放问题

| # | 模块 | 问题 | AI 猜测 | 答案 |
|---|------|------|---------|------|
| 1 | 用户体验 | 用户期望的音频效果是什么样的？是实时播放还是导出为带音频的视频文件？ | 导出为带音频的 mp4 视频 | |
| 2 | 背景音乐 | BGM 的来源是什么？AI 生成、预置音乐库、还是用户上传？ | 预置免费音乐库 + 按场景自动选择 | |
| 3 | 配音 | 配音用什么方案？本地 TTS 引擎、在线 TTS API（如 Edge TTS）、还是 AI 语音合成？ | Edge TTS（免费、中文效果好） | |
| 4 | 角色区分 | 多个角色对话时，是否需要不同音色区分不同角色？ | 需要，至少男/女两种音色 | |
| 5 | 业务规则 | 音频是否需要和动画帧精确同步（角色说话时嘴动）？还是简单叠加即可？ | 简单叠加，台词出现时播放对应语音 | |
| 6 | 范围影响 | 这个功能是否影响现有的纯视觉预览模式？用户能否选择静音/有声？ | 不影响，默认有声，可静音 | |
