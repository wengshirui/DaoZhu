# 048 — 用户手动绑定文件夹为工作区

**Intake ID:** 2026-05-28-manual-folder-workspace
**Requester:** PO
**Urgency:** Normal
**Status:** 待澄清

---

## 原始需求

> 现在工作区都是 AI 创建的，增加允许用户手动上传文件夹作为工作区。点击预览就是打开资源管理器。也就是工作区增加一种类型，是用户创建，只要绑定到本地文件夹就行。

## 提取意图

当前工作区只能通过 AI Agent 创建（生成代码 + workspace.json + .venv 等），用户无法将已有的本地项目/文件夹纳入岛主管理。需要新增一种"绑定型"工作区：

- 不需要 AI 生成代码，只需指向一个本地文件夹路径
- 在书架上和其他工作区一样展示（图标、名称、状态）
- 点击"预览/打开"时，直接打开系统资源管理器到该文件夹
- 用户自己管理内容，岛主只做入口和索引

## 开放问题

| # | 模块 | 问题 | AI 猜测 | 答案 |
|---|------|------|---------|------|
| 1 | 用户体验 | 用户绑定文件夹的交互方式是什么样的？比如：弹窗选择文件夹、拖拽文件夹到书架、还是输入路径？ | 弹窗选择文件夹（系统原生目录选择器） | |
| 2 | 范围 | 绑定后的工作区是否需要支持启动/停止（如果文件夹里有 app.py）？还是纯粹只做文件夹快捷入口？ | 纯快捷入口，不管理进程 | |
| 3 | 展示 | 绑定型工作区在书架上如何和 AI 创建的工作区区分？需要不同图标/标签吗？ | 显示一个"📁"图标或"本地"标签区分 | |
| 4 | 元数据 | 用户是否需要为绑定的文件夹设置名称、图标、颜色等自定义信息？ | 需要，至少支持自定义名称和图标 | |
| 5 | 生命周期 | 如果用户删除/移动了绑定的文件夹，岛主如何处理？ | 显示"路径不存在"警告，允许重新绑定或移除 | |
| 6 | 数量限制 | 用户可以绑定多少个文件夹？是否有上限？ | 无上限，和 AI 工作区混排显示 | |

**S (3pts)** — 纯前端交互 + 少量后端 API（保存路径、打开资源管理器），不涉及进程管理和复杂业务逻辑。唯一复杂点是系统文件夹选择器在 Web 端的实现方式。
