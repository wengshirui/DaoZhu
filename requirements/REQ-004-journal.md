# REQ-004 做账（分录生成/审核/过账）

**状态：** ✔️ Done  
**优先级：** P0（核心业务流程）  
**T-Shirt Size：** L - AI 智能匹配科目是核心难点，需要领域知识 + 用户反馈学习

---

## 问题陈述

会计分录是将经济业务翻译为"借贷记账法"的过程。传统做法需要会计人员判断该借什么科目、贷什么科目。AccoBot 的核心价值是让 AI 自动完成这个判断，用户只需确认或修正。

## 用户故事

**As a** 用户  
**I want** 告诉 AI 发生了什么业务，AI 自动生成会计分录  
**So that** 我不需要记住复杂的科目对应关系，也不会做错分录

## 范围

### In Scope

- **智能分录生成** — 根据凭证摘要/业务描述，AI 自动建议借贷科目和金额
- **科目智能匹配** — 基于业务关键词、历史分录模式、行业规则匹配科目
- **分录编辑** — 用户可修改 AI 建议的分录（改科目、改金额、加行）
- **借贷平衡校验** — 自动检查借方合计 = 贷方合计
- **凭证审核** — 审核通过后锁定，不可随意修改
- **过账** — 将已审核凭证写入账簿
- **期末结转** — 损益类科目自动结转到本年利润
- **红冲/反过账** — 错误凭证的更正机制

### Out of Scope

- 原始凭证录入（REQ-003）
- 账簿查询（REQ-007）

## 验收标准

1. **AC-1：智能建议** — 用户输入"付了5000元房租"，Agent 建议分录：借-管理费用-租赁费 5000 / 贷-银行存款 5000，用户确认即可保存
2. **AC-2：复合分录** — 支持一借多贷、多借一贷、多借多贷的复合分录
3. **AC-3：借贷平衡** — 保存分录时自动校验借贷平衡，不平衡时拒绝保存并提示差额
4. **AC-4：辅助核算** — 如果科目挂了辅助核算，生成分录时自动提示用户选择（如：选择部门）
5. **AC-5：学习能力** — 用户修正过的分录模式被记忆，下次类似业务优先使用修正后的模式
6. **AC-6：期末结转** — 用户说"帮我做期末结转"，Agent 自动将所有损益科目余额结转到本年利润，生成结转凭证
7. **AC-7：红冲** — 用户说"上个月那笔房租分录做错了"，Agent 生成红字冲销凭证 + 正确的蓝字凭证

## 业务规则

- 每笔分录至少有一个借方和一个贷方
- 借方合计必须等于贷方合计（允许分差 ≤ 0.01 元自动调整）
- 只有末级科目才能录入分录
- 已关闭期间不允许新增/修改分录
- 过账操作不可逆（只能通过红冲更正）
