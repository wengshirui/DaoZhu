# REQ-022 Skill 透明化自动学习

**状态：** 📝 Draft  
**优先级：** M  
**来源：** 2025-05-21 架构 Grill 讨论

---

## 问题陈述

当前 Skill 创建需要用户明确要求"保存为 Skill"。但目标用户（小微老板）不理解 Skill 概念，也不会主动触发保存。需要 Agent 在对话中自动识别"值得记住的操作流程"，透明地创建 Skill，下次遇到类似场景自动应用。用户无需知道 Skill 的存在。

## 范围

**In Scope:**
- Agent 自动识别可复用的操作流程（如：用户纠正了 Agent 的操作方式）
- 自动创建 Skill，无需用户确认（对用户透明）
- 下次遇到类似场景时自动加载并应用已有 Skill
- Agent 被用户纠正操作后，主动提议保存（"要不要我记住这个流程？"）

**Out of Scope:**
- 用户手动管理 Skill（列表/编辑/删除 UI）——0.1 不需要
- Skill 版本控制或冲突合并
- 跨账套的 Skill 共享

## 用户故事

As a 小微企业老板，I want AccoBot 能从我的纠正中学习，下次自动做对，so that 我不需要重复教它同样的事情。

## 验收标准

1. **AC-1 纠正检测**：当用户纠正了 Agent 的操作方式（如"不对，应该记到管理费用不是销售费用"），Agent 识别到这是一次纠正
2. **AC-2 主动提议**：检测到纠正后，Agent 主动询问"要不要我记住这个规则，下次自动这样做？"
3. **AC-3 透明创建**：用户同意后（或 Agent 判断为通用规则时），自动创建 Skill，不向用户暴露 Skill 的技术概念（不说"已创建 Skill"，而说"好的，我记住了"）
4. **AC-4 自动应用**：后续对话中遇到匹配场景时，Agent 自动加载相关 Skill 并按其流程执行，无需用户再次指示
5. **AC-5 基于实际操作**：自动创建的 Skill 内容必须基于本次对话中实际执行过的操作步骤，不得臆想或编造

## 业务价值

- 持续提升准确率：每次纠正都让系统变得更好，形成正向飞轮
- 降低用户重复劳动：同类业务不需要每次都解释
- 差异化竞争力：传统财务软件无法"学习"用户偏好

## T-Shirt Size

**M** - 需要在 Agent 对话循环中增加"纠正检测"逻辑 + Skill 自动创建 + 场景匹配加载；风险在于纠正检测的准确率（误判会创建错误 Skill）

## 依赖与约束

- 依赖 REQ-016（Skill 系统基础设施）
- 依赖 REQ-012（记忆系统，用于判断是否已有相关 Skill）
- 约束：误创建的 Skill 不应导致后续操作错误——需要 Skill 应用时仍经过质检（REQ-021）
- 约束：Skill 创建不应打断对话流程
