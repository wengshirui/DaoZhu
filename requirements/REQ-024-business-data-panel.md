# REQ-024 业务操作区（数据浏览与筛选）

**状态：** 📝 Draft  
**优先级：** M  
**来源：** 2026-05-22 用户需求

---

## 问题陈述

当前用户查看财务数据（科目、凭证、分录、账簿、报表）只能通过对话让 Agent 查询，不够直观。用户希望像传统财务软件一样直接翻看和筛选数据。需要将现有 Web UI 的面板改造为"业务操作区"，按类别展示核心财务数据，支持点击查看和条件筛选。同时将现有的待办提醒集中到统一入口。

## 范围

**In Scope:**
- 将现有面板改造为"业务操作区"，按类别（科目/辅助属性/凭证/分录/账簿/报表）展示数据
- 每类数据支持列表浏览 + 条件筛选
- 筛选维度：日期、状态、科目、金额范围、摘要关键字
- 点击条目展开详情
- 待办提醒集中到统一入口（如顶部通知图标或独立 tab），不再占用主面板空间

**Out of Scope:**
- 数据的直接编辑（编辑仍通过对话让 Agent 操作）
- 复杂的报表可视化（图表等）
- 导出功能（Excel/PDF）
- 移动端适配

## 用户故事

As a 小微企业老板，I want 在界面上直接点击查看凭证、科目、账簿等财务数据并按条件筛选，so that 我不需要每次都通过对话问 Agent 才能看到自己的账务情况。

## 验收标准

1. **AC-1 业务操作区入口**：Web UI 面板改造为"业务操作区"，包含按类别分组的导航（科目、辅助属性、凭证、分录、账簿、报表）
2. **AC-2 数据列表**：点击类别后展示对应数据的列表视图，凭证显示（日期/编号/摘要/金额/状态），科目显示（编码/名称/类别/余额方向），账簿显示（科目/期间/借方合计/贷方合计/余额）
3. **AC-3 筛选功能**：每类数据至少支持 2 个筛选维度——凭证（日期范围 + 状态）、科目（类别）、账簿（科目 + 期间）、分录（摘要关键字 + 金额范围）
4. **AC-4 详情展开**：点击列表中的条目可展开查看完整详情（如凭证的所有分录明细）
5. **AC-5 待办集中**：原面板中的待办提醒移至统一入口（如顶部通知区域），点击后按类型分组显示（税务提醒/凭证待审/系统通知）
6. **AC-6 无数据状态**：未选择账套或账套无数据时，显示友好的空状态提示和引导

## 业务价值

- 降低使用门槛：不懂如何描述查询需求的用户也能直接浏览数据
- 提升效率：查看凭证列表从"对话 3 轮"变为"点击 1 次"
- 增强信任感：用户能直接看到数据，确认 Agent 操作的结果正确
- 产品体验对齐：接近传统财务软件的操作习惯，降低迁移心理成本

## T-Shirt Size

**M** - 前端工作量较大（6 类数据的列表/筛选/详情），后端 API 大部分已有（REQ-010 已实现部分）；风险在于纯 vanilla JS 实现复杂表格+筛选的维护成本

## 依赖与约束

- 依赖 REQ-010（Web UI 基础框架、现有 API）
- 依赖 REQ-002（科目/辅助核算数据）
- 依赖 REQ-003/004（凭证/分录数据）
- 依赖 REQ-007（账簿数据）
- 依赖 REQ-008（报表数据）
