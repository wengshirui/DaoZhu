# REQ-023 账套会计准则维护

**状态：** 📝 Draft  
**优先级：** M  
**来源：** 2026-05-22 用户需求

---

## 问题陈述

当前账套创建时选择了会计准则（如 small_enterprise），但准则的具体要求（标准科目表、业务规则、核算约束）没有结构化记录在账套目录下。Agent 做账时缺乏明确的准则约束参考，可能生成不符合所选准则的分录。需要将准则要求文档化并持久化到账套文件中，供 Agent 读取作为做账约束。

## 范围

**In Scope:**
- 账套创建/编辑时可选择或切换会计准则
- 准则选择后，在账套目录下生成准则要求文档（标准科目表、核算规则、业务约束）
- 准则要求文档供 Agent 读取，作为做账时的约束规则
- 不同准则对应不同的标准科目表（如小企业准则无"递延所得税"科目）
- 切换准则只影响后续操作，历史数据不动

**Out of Scope:**
- 历史凭证/科目的自动迁移转换
- 自定义准则（仅支持国家标准准则）
- 准则文档的在线编辑（由系统预置）

## 用户故事

As a 小微企业老板，I want 账套能明确关联一套会计准则，Agent 做账时严格遵守该准则的规则，so that 我的账务处理符合国家规定，不会因为准则适用错误被税务局质疑。

## 验收标准

1. **AC-1 准则选择**：创建账套时必须选择会计准则（小企业会计准则 / 企业会计准则），选择后记录到账套元数据中
2. **AC-2 准则文档生成**：准则选择后，在账套目录下生成结构化文档（如 `standard/accounting_rules.md` 和 `standard/chart_of_accounts.json`），包含该准则的标准科目表和核算规则
3. **AC-3 Agent 约束读取**：Agent 做账时能读取当前账套的准则文档，生成分录时遵守准则约束（如小企业准则下不使用"递延所得税资产"科目）
4. **AC-4 准则切换**：支持修改账套的会计准则，切换后重新生成准则文档，已有凭证和科目不受影响，仅后续操作按新准则执行
5. **AC-5 切换提示**：切换准则时向用户说明影响范围（"切换后新增凭证将按企业会计准则处理，已有数据不变"）

## 业务价值

- 合规保障：不同规模企业适用不同准则，错误适用可能导致税务风险
- 提升 Agent 准确率：有明确约束文档，减少模型"自由发挥"导致的科目误用
- 为多账套场景打基础：代账公司不同客户可能适用不同准则

## T-Shirt Size

**M** - 需要设计准则文档结构、预置两套标准科目表/规则、改造账套创建流程、Agent system prompt 动态加载准则；风险在于准则规则的完整性和准确性

## 依赖与约束

- 依赖 REQ-002（账套管理基础）
- 约束：初期支持两种准则——小企业会计准则、企业会计准则
- 约束：准则文档由系统预置，不支持用户自定义修改
