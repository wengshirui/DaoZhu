# REQ-015 质检与风控

**状态：** ✔️ Done  
**优先级：** P1（财务安全的核心保障）  
**T-Shirt Size：** L - 规则体系庞大，需要持续更新，AI 判断 + 规则引擎结合

---

## 问题陈述

财务工作的错误代价高昂：轻则账目混乱需要返工，重则违法违规面临罚款甚至刑事责任。传统做法依赖人工审核，但人会疲劳、会遗漏。AccoBot 应该在每一笔操作时自动进行风险审查，像一个永不疲倦的"质检员"。

## 用户故事

**As a** 用户  
**I want** AI 在我做账时自动检查是否有风险，主动告诉我哪里有问题  
**So that** 我不会因为不懂规则而犯错，也不会因为疏忽而遗漏风险

## 范围

### In Scope

**企业风控：**
- 超预算支出预警
- 异常大额交易提醒
- 关联交易识别
- 费用报销超标检查
- 现金流异常（大额现金收支）
- 往来款项长期挂账

**国家政策合规：**
- 会计准则符合性检查（科目使用是否正确、核算方法是否恰当）
- 会计期间匹配（是否跨期入账）
- 资产折旧/摊销是否按规定执行

**法律合规：**
- 涉税风险（虚开发票特征、进项抵扣不合规、收入确认时点）
- 反洗钱规则（大额现金交易、频繁小额转账）
- 资金往来合规性（公转私、私转公的合理性）

**数据一致性：**
- 借贷不平衡
- 重复录入检测
- 凭证断号
- 科目余额方向异常（如：资产类科目出现贷方余额）

### Out of Scope

- 自动修正（只提示风险，不自动改动用户数据）
- 法律咨询服务（只提示风险点，不提供法律建议）
- 审计报告出具

## 风险分级

| 级别 | 标识 | 含义 | 处理方式 |
|------|------|------|---------|
| 严重 | 🔴 | 可能违法违规，必须处理 | 阻断操作，要求用户确认后才能继续 |
| 警告 | 🟡 | 存在风险，建议处理 | 提示用户，不阻断操作 |
| 提示 | 🟢 | 最佳实践建议 | 轻提示，用户可忽略 |

## 验收标准

1. **AC-1：实时检查** — 用户保存凭证/分录时，自动触发质检，1 秒内返回检查结果
2. **AC-2：风险分级** — 检查结果按严重/警告/提示三级分类展示，严重级别高亮显示
3. **AC-3：阻断机制** — 严重风险（如借贷不平衡）阻止保存，用户必须修正后才能继续
4. **AC-4：规则可配** — 企业风控规则（如报销上限、大额标准）可由用户自定义配置
5. **AC-5：解释说明** — 每条风险提示附带原因说明和法规依据（如："根据《增值税暂行条例》第X条……"）
6. **AC-6：批量质检** — 用户说"帮我检查一下这个月的账"，Agent 对当月所有凭证进行全面质检并生成报告
7. **AC-7：规则更新** — 税收政策变化时，通过更新 Skill 文件即可生效，无需改代码

## 质检规则示例

```yaml
# 企业风控规则（用户可配置）
rules:
  - name: 大额现金交易
    condition: "cash_amount > 50000"
    level: warning
    message: "单笔现金交易超过5万元，请确认是否合规"
    
  - name: 费用报销超标
    condition: "expense_category == '差旅费' and amount > config.travel_limit"
    level: warning
    message: "差旅费超出企业标准上限"

  - name: 借贷不平衡
    condition: "sum(debit) != sum(credit)"
    level: critical
    message: "借贷不平衡，差额 {diff} 元"
```
